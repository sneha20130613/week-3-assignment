4. Development Process

The development started by outlining the content and structure for each page. Semantic HTML5 elements were used to build the basic framework. An external CSS file (style.css) was then created to apply styling and ensure responsiveness across all pages. The design focused on clarity, ease of navigation, and a consistent look and feel. Iterative testing on different screen sizes was performed to refine the responsive behavior.

5. Responsiveness and Styling

The website was made responsive using CSS. Key techniques include:

  • Viewport Meta Tag: The tag is included in all HTML files to ensure proper scaling and rendering on mobile devices.
  • Fluid Layouts: Using percentages and flexible box (flexbox) for elements like the navigation bar to adapt to screen width. Main content areas use a `max-width` and `margin: auto` for centering and adaptability.
  • Media Queries: CSS media queries are used to apply different styles based on screen size. For example, the navigation bar stacks vertically on smaller screens (tablets and mobiles) for better usability. Font sizes and padding are also adjusted.
  • Responsive Images: Images are styled with max-width: 100%; height: auto; to ensure they scale down proportionally within their containers without distortion.
  • External CSS: All styling is managed through an external stylesheet (style.css). This includes global styles for typography, colors, and layout, as well as specific styles for different sections and pages (e.g., the background image on the contact page, project article styling).

6. GitHub Pages Deployment

The portfolio website is hosted on GitHub Pages. The deployment process involved the following steps:

  1. Create a GitHub Repository: A new public repository named "portfolio" was created on my GitHub account (github.com/YOUR_USERNAME).
  2. Initialize Git and Add Project Files: A local Git repository was initialized in the project folder. All HTML files (index.html, about.html, projects.html, contact.html, report.html), the CSS file (style.css), and any images (e.g., the background image for the contact page) were added to the staging area.
  3. Commit and Push Files: The files were committed to the local Git repository. The local repository was then linked to the remote "portfolio" repository on GitHub, and the `main` branch was pushed. 
    git init
git add .
git commit -m "Initial commit of responsive portfolio website"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/portfolio.git
git push -u origin main
    (Replace `YOUR_USERNAME` with your actual GitHub username.)
  4. Enable GitHub Pages:
    • Navigated to the "portfolio" repository on GitHub.
    • Clicked on the "Settings" tab.
    • In the left sidebar, selected "Pages" under the "Code and automation" section.
    • Under "Build and deployment," for "Source," selected "Deploy from a branch."
    • Under "Branch," selected "main" (or your default branch) and the /(root) folder.
    • Clicked "Save."
  5. Access Live Site: After a few minutes for GitHub to build and deploy the site, it became available online. The live link is updated periodically if changes are pushed.
